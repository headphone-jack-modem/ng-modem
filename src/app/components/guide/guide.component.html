layers Overview

Communicate between microcontroller and mobile via headphone jack.

There is one library for microcontrollers, specialized for ATmega16 written with C, here; and another library for smart devices (such as laptop, Android, or iPhone mobiles) that runs on the browser platforms (such as firefox), here.

Using these two libraries and simple hardware, you can connect a microcontroller to a smart device via the headphone jack and then communicate between them.

tips_and_updates How It Works

It works based on th BFSK modulation. At the time of transmission, the bits of each byte is converted to high or low frequencies. At the beginning of each byte, a lower bit is added to display the beginning of the byte, and at the end of the byte, three more bits are added, including the following:

  • Control bit: Indicates that the byte is metadata or data.
  • Preity bit:
  • Stop bit: It's a high bit to indicates the end of the byte

And each message transfer contains one byte of metadata that holds the number of bytes of the message.

For example, to send a message containing "OK" (ie 01001111 01001011 in binary), we add a metadata byte that holds the size of the transmitted message (ie 00000010 in binary). Then we add the start, control, parity, and stop bits to each of these bytes.
